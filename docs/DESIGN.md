# Autorun Harness Dashboard 设计规范

## 设计原则

1. **简洁优先** - 界面简洁，信息层次分明
2. **实时反馈** - 状态变化即时呈现
3. **一目了然** - 关键信息一眼可见

## 颜色系统

### 主色调
- **Primary (蓝色)**: `#3b82f6` - 主要操作、链接
- **Success (绿色)**: `#22c55e` - 成功状态、已完成
- **Warning (黄色)**: `#f59e0b` - 警告、阻塞
- **Danger (红色)**: `#ef4444` - 错误、需人工介入

### 任务状态颜色
| 状态 | 背景 | 文字 |
|------|------|------|
| pending | gray-100 | gray-700 |
| in_progress | blue-100 | blue-700 |
| completed | green-100 | green-700 |
| blocked | yellow-100 | yellow-700 |
| needs_human | red-100 | red-700 |

## 组件规范

### 按钮
- **Primary**: `bg-primary-500 hover:bg-primary-600 text-white`
- **Secondary**: `bg-gray-100 hover:bg-gray-200 text-gray-700`
- **Danger**: `bg-red-500 hover:bg-red-600 text-white`
- **尺寸**: `px-4 py-2 rounded-md text-sm font-medium`

### 卡片
- `bg-white rounded-lg shadow-sm border border-gray-200 p-4`

### 标签
- 小标签: `px-2 py-0.5 text-xs rounded-full`
- 颜色根据状态变化

### 输入框
- `border border-gray-300 rounded-md px-3 py-2 focus:ring-2 focus:ring-primary-500 focus:border-primary-500`

## 图标

使用简洁的符号图标：
- ✓ 完成
- ✗ 失败
- ○ 待处理
- ● 进行中
- ⚠ 警告
- ⚡ 需人工介入

## 动画

- 状态变化: `transition-colors duration-200`
- 进度条: `transition-all duration-500`
- 新项目添加: `animate-fade-in`

## 响应式

- 移动端 (< 640px): 单列布局
- 平板 (640px - 1024px): 双列布局
- 桌面 (> 1024px): 三列布局或看板四列
